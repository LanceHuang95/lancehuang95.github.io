4 栈和队列

栈(stack):限定仅在表尾进行插入和删除操作的线性表，通常把允许插入和删除的一端称为栈顶top;栈又称为先进后出(LIFO)的线性表

  • 栈是一个线性表
  • 栈的特殊之处在于限定了线性表的插入和删除位置，始终在栈顶(表尾)进行插入和删除
  • 栈的栈底是固定的，第一个进入栈的元素在栈底

栈的抽象数据类型

ADT 栈(stack)
Data
    同线性表.元素具有相同的类型，相邻元素具有前驱和后继关系
Operation
    InitStack(*s)
    DestoryStack(*s)
    ClearStack(*s)
    StackEmpty(s)
    GetTop(s,*e)
    Push(*s,e)
    Pop(*s,*e)
    StackLength(s)
endADT

1.栈的顺序存储结构

  • 栈底：数组下标为0的一端，
  • 栈顶：数组高下标的位置，因为在插入和删除时不需要移动数组中的元素

### 2.两个栈共享空间

用一个数组存储两个栈，一个栈的栈底为数组下标为0处(数组始端)，另一个栈的栈底为下标n-1处(数组末端),两个栈分别向中间靠拢
(见大话数据结构Page95 图4.5.1)

假设top1和top2分别为栈1和栈2的栈顶，当top1=-1时，栈1为空，top2=n时，栈2为空;
易知top1+1==top2时栈满
```cpp
/*两栈共享数组空间*/
typedef struct SequenceDoubleStack
{
    ElemType data[MaxSize];
    int top1;
    int top2;
}SDStack;

/*插入元素为e的新的栈顶元素,stackNum判断是栈1还是栈2的参数*/
int Push(SDStack *S,ElemType e,int stackNum)
{
    if(S->top1+1 == S->top2)
        return false;
    if(stackNum ==1)
        S->data[++S->top1]=e;
    else if(stackNum ==2)
        S->data[--S->top2]=e;
    return true;
}

/*删除栈顶元素,并用e返回其值*/
int Pop(SDStack *S,ElemType *e,int stackNum)
{
    if(stackNum ==1)
    {
        if(S->top1 ==-1)
            return false;
        *e=S->data[S->top1--];
    }
    else if (stackNum ==2)
    {
        if(S->top2 ==n)
            return false;
         *e=S->data[S->top2++];
    }
    return true;
}

这样的数据结构适用于同元素类型的栈,特别是一个栈增长一个栈缩短时

3.栈的链式存储结构

栈的链式结构：由于栈需要栈顶，单链表有头指针，故可以把栈顶放在单链表的头部，对于链栈来说，此时头节点就无足轻重了(栈的操作都是基于栈顶)。一个空的链栈就是top=NULL

/*链栈的结构*/
typedef struct StackNode
{
    ElemType data;
    struct StackNode * next;
}SNode;

typedef struct LinkStack
{
    SNode * top;//栈顶指针
    int length;
}LStack;

链栈的进栈和出栈操作

/*插入元素为e的新的栈顶元素,类似于头插法*/
int Push(LStack * S,ElemType e)
{
    SNode * new=(SNode *)malloc(sizeof(SNode));
    new->data=e;
    new->next=S->top;//进栈

    S->top=new;//修改栈顶
    S->length++;
    return true;
}

/*删除栈顶元素,保存到元素e中*/
int Push(LStack * S,ElemType *e)
{
    if(StackEmpty(*S))
        return false;

    *e=S->top->data;
    SNode * pCurr=S->top;
    S->top=pCurr->next//出栈

    S->length--;

    free(pCurr);
    return true;
}

总结：

  • 顺序栈和链式栈的进栈和出栈在时间复杂度都为O(1)
  • 顺序栈需要确定固定的长度，可能存在内存空间浪费的问题,优点是读取定位方便；链式栈每个元素都有指针域也同样增加了内存开销，但栈的长度无限制
  • 如果栈的长度确定，顺序栈更好；若栈的长度不能确定，链式栈更好

4.栈的应用

递归

递归和迭代的区别:

  • 迭代使用循环结构，递归使用选择结构
  • 递归结构更简洁易懂，但递归的调用会建立函数副本，会耗费大量时间和内存

编译器使用栈实现递归：在前行阶段，对于每一层递归，函数的局部变量、参数及返回地址都被压入栈中；在退回阶段，位于栈顶的局部变量、参数及返回地址都被弹出，用于返回调用层次中执行代码的其余部分，即恢复调用状态

四则运算求表达式

中缀表达式：目前我们习惯的写法
后缀表达式：运算符放数字后面的写法，计算机计算

  1. 中缀表达式转后缀表达式：

    数字：直接输出

括号和运算符

  • 左括号：进栈
  • 运算符：与栈顶符号进行优先级比较
    • 若优先级不低于栈顶符号，此符号进栈（乘除高于加减）
    • 若优先级低于栈顶符号，则将栈顶符号弹出，之后进栈
  • 右括号：将栈顶符号弹出，直到匹配左括号
    按顺序输出则为后缀表达式
/*中缀表达式转后缀表达式伪代码*/
void transform(exp)
{
    创建栈S;
    i=0;
    while(exp[i] != '\0')
    {
        if(exp[i]为数字)
            Output(exp[i]);
        else if(exp[i]为符号)
        {
            while(exp[i]优先级<=栈顶符号优先级)
                {
                    Output(栈顶符号)；
                    Pop(S);
                }
                Push(S,exp[i]);
        }
        else if(exp[i]为左括号)
            Push(S,exp[i]);
        else if(exp[i]为右括号)
        {
            while(栈顶不为左括号)
            {
                Output(栈顶符号)；
                Pop(S);
            }
            Pop(左括号);
        }
        else
        {
            错误，停止循环;
        }

        i++
    }
    while(size(S) > 0 && exp[i] == ‘\0’)
    {
        output(栈顶符号)；
        Pop（S）;
    }

}

  • 对于数字：进栈

  • 遍历结束：输出栈中唯一数字

### 5.队列
队列：只允许一端进行插入(队尾)，另一端进行删除(对头)的线性表，是一种先进先出(FIFO)的线性表
```cpp
/*队列的抽象数据结构*/
ADT 队列(Queue)
Data
    同线性表。元素具有相同数据类型，相邻元素具有前驱和后继关系
Operation
    InitQueue(*Q)
    DestoryQueue(*Q)
    ClearQueue(*Q)
    QueueEmpty(Q)
    GetHead(Q,*e)
    EnQueue(*Q,e)//插入队尾
    DeQueue(*Q,*e)//删除队头
    QueueLength(Q)
endADT

队列的顺序存储

假设数组的下标为0的一端为队头，另一端为队尾

  • 入队(EnQueue)操作就是在队尾添加一个元素，不需要移动，时间复杂度O(1)
  • 出队(DeQueue)操作就是在队头删除元素，即下标为0的位置元素删除，其余元素往前移动一个位置，时间复杂度O(n),代码实现参考顺序线性表
  • 假设增加队头指针，则出队时不用移动元素位置，只需改变队头指针；但队头指针前的空间位置浪费，假溢出，则引入循环队列(见大话数据结构Page114 图4.12.5)

6.顺序循环队列

  • 引入front指针指向队头元素，rear指针指向队尾元素的下一个位置
  • 当为空队列时，front等于rear；当队列满时，front也等于rear,那么如何判断队列究竟是空还是满
    • 方法一：设置一个标志位flag
    • 方法二：当队列满时，修改条件，保留一个元素位置空间，此时认为队列为满(见大话数据结构Page115 图4.12.8)

对于方法二,队列满的条件为(rear+1)%QueueSize == front ;队列的长度为(rear-front+QueueSize)%QueueSize

循环队列的顺序存储结构即初始化

/*循环队列的顺序存储结构*/
typedef int ElemType;
typedef struct CircleSequenceQueue
{
    ElemType  data[MaxSize];
    int front; //头指针
    int rear; //尾指针，若队列不为空，指向队尾元素的下一个位置
}CSQueue;

int InitQueue(CSQueue *Q)
{
    Q->front =0;
    Q->rear=0;
    return true;
}

int QueueLength(CSQueue Q)
{
    return (Q.rear-Q.front+MaxSize)%MaxSize;
}

循环队列的入队出队


/*循环队列的顺序存储结构*/
int EnQueue(CSQueue *Q,ElemType e)
{
   if((Q->rear+1)%MaxSize == Q->front)
       return false;
    Q->data[Q->rear] =e;
    Q->rear=(Q->rear+1)%MaxSize;//rear指针后移一位，若到最后则转到数组头部
    return true;
}

int DeQueue(CSQueue *Q,ElemType *e)
{
   if(Q->front == Q->rear)
       return false;
    *e=Q->data[Q->front];
    Q->front=(Q->front+1)%MaxSize;
    return true;
}

7.队列的链式存储结构即实现

链式队列：队头为第一个节点，队尾为终端节点；为方便操作，让队头指针指向头节点，空队列时，front和rear都指向头节点

/*链式队列结构*/
typedef int ElemType;
typedef struct QueueNode
{
    ElemType data;
    struct QueueNode * next;
}QNode;

typedef struct LinkQueue
{
    Queue* front,rear;//队头对指针
    //其他需要元素自己添加
}LQueue;

链式队列入队出队操作

/*入队，在链表尾部插入节点*/
int EnQueue(LQueue *Q,ElemType e)
{
    QNode * new =(QNode *)malloc(sizeof(Node));
    new->data=e;
    new->next=NULL;

    Q->rear->next=new;//尾部插入节点
    Q->rear=new;//更新尾部节点

    return true;
}

/*出队，删除链表第一个节点*/
int DeQueue(LQueue *Q,ElemType *e)
{
    if(Q->front == Q->rear)
        return false;

    Node * pCurr=Q->front->next;//保存第一个节点
    *e=pCurr->data;

    Q->front->next=pCurr->next;//出队，更新第一个节点

    if(Q->rear == pCurr)//若队列只有一个元素，队头就是队尾时，出对后将rear指向头节点
        Q->rear=Q->front;

    free(pCurr);
    return true;
}
